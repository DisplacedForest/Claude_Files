# GitHub Issue Update Template (for /plan command)

## Feature Issue Format

```markdown
## Overview
{Brief description of the feature and its importance}

## Problem Statement
{Detailed description of the problem this solves}

## User Flow
### Current State (if applicable)
- Step 1: {current behavior}
- Step 2: {pain point}
- Step 3: {user frustration}

### Desired State
- Step 1: {improved behavior}
- Step 2: {smooth flow}
- Step 3: {user satisfaction}

## Success Criteria
- [ ] {Specific, measurable outcome 1}
- [ ] {Specific, measurable outcome 2}
- [ ] {Specific, measurable outcome 3}
- [ ] {User-facing success metric}
- [ ] {Technical success metric}

## Technical Requirements
### Backend
- [ ] {API endpoint requirement}
- [ ] {Database schema change}
- [ ] {Integration requirement}
- [ ] {Performance requirement}

### Frontend
- [ ] {UI component requirement}
- [ ] {State management requirement}
- [ ] {User interaction requirement}
- [ ] {Responsive design requirement}

### Infrastructure
- [ ] {Deployment requirement}
- [ ] {Monitoring requirement}
- [ ] {Security requirement}

## Implementation Plan
### Phase 1: {Foundation}
- [ ] {Task 1}
- [ ] {Task 2}
- [ ] {Task 3}

### Phase 2: {Core Functionality}
- [ ] {Task 1}
- [ ] {Task 2}
- [ ] {Task 3}

### Phase 3: {Polish & Edge Cases}
- [ ] {Task 1}
- [ ] {Task 2}
- [ ] {Task 3}

## Acceptance Tests
- [ ] Test: {Scenario 1}
  - Given: {context}
  - When: {action}
  - Then: {expected result}
- [ ] Test: {Scenario 2}
  - Given: {context}
  - When: {action}
  - Then: {expected result}

## Business Impact
### Why This Matters
{Explanation of business value}

### Key Metrics
- {Metric 1}: Expected impact
- {Metric 2}: Expected impact
- {Metric 3}: Expected impact

## Dependencies
- **Blocker**: {Critical dependency}
- **Required**: {Necessary dependency}
- **Nice-to-have**: {Optional dependency}

## Risks & Mitigations
| Risk | Impact | Probability | Mitigation |
|------|--------|-------------|------------|
| {Risk 1} | High | Medium | {Mitigation strategy} |
| {Risk 2} | Medium | Low | {Mitigation strategy} |

## Open Questions
- [ ] {Question needing answer before implementation}
- [ ] {Design decision to be made}
- [ ] {Technical investigation needed}

## Definition of Done
- [ ] All success criteria met
- [ ] Tests written and passing
- [ ] Documentation updated
- [ ] Code reviewed and approved
- [ ] Deployed to production
- [ ] Metrics tracking in place
- [ ] User feedback collected

---
*Spec generated by `/plan` command on {date}*
```

## Bug Issue Format (for reference)

```markdown
## Bug Description
{Clear description of what's broken}

## Current Behavior
{What actually happens}
- Step 1: {action}
- Step 2: {unexpected result}
- Impact: {user/system impact}

## Expected Behavior
{What should happen}
- Step 1: {action}
- Step 2: {expected result}

## Root Cause Investigation
- [ ] {Investigation task 1}
- [ ] {Investigation task 2}
- [ ] {Investigation task 3}
- [ ] {Investigation task 4}

## Reproduction Steps
1. {Detailed step}
2. {Detailed step}
3. {Observe bug}

## Environment
- **Affected Environments**: {prod/staging/dev}
- **Browser/OS**: {details}
- **User Reports**: {number}
- **First Occurrence**: {date}

## Technical Analysis
### Potential Causes
- [ ] {Hypothesis 1}
- [ ] {Hypothesis 2}
- [ ] {Hypothesis 3}

### Required Fixes
- [ ] {Fix component 1}
- [ ] {Fix component 2}
- [ ] {Add error handling}
- [ ] {Add tests}

## Success Criteria
- [ ] Bug no longer reproducible
- [ ] Regression tests added
- [ ] Error handling improved
- [ ] Monitoring added for issue
- [ ] Documentation updated

## Priority Justification
{Why this needs immediate attention}

## Dependencies
- {Dependency 1}
- {Dependency 2}

---
*Bug report created on {date}*
```
